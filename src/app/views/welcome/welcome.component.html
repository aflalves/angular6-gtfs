About this project

Welcome!

This project is part of a coding challenge, and my task was to build a frontend application featuring a GTFS dataset along with a map to display meaningful information.

The documentation below explains some of the main ideas and thoughts for this project.

The GTFS and the GtfsService

The GTFS source is from {{agency.agency_name}} and it is currently stored as json (converted using csvjson) on a mock file, which is retrieved by an Angular Service ( GtfsService) that exposes GTFS data to the application. This abstraction makes the process of swapping the mocked json to a real API retrieved json completely invisible for the Angular UI components that uses the GTFS dataset from the GtfsService.

Trip Component

The TripComponent listens do an event from the child LinesComponent that sends the trip_id of a trip initally retrived from the GTFS trips.txt. The TripComponent will then ask the GtfsService to build the trip data, which consists of two steps:

- Get all the stops of this trip from GTFS stop_times.txt by trip_id;
- Add the stop data (coordinates, description, etc) to each of those stops using a hash map.

With the trip data completed, the Angular Google Maps will render all the stops with their arrival time, description and draw polylines to connect them. This screen could be improved if we use data from shapes.txt, but that requires more effort to be implemented, although it would be very nice to draw the exact path of the bus.

Lines Component

The LinesComponent lists one random trip of each route and it is used by the TripComponent. It doesn't show all trips because the mocked GTFS from trips.txt doesn't contain every trip. The ideia was to reduce the complexity of having to deal with the recurrence of the same route on different times/days, letting us focus on showing each route currenly available. An improvement I would work on is to show all recurrences of each route, on different days and times. User could select a specific trip of the route and see it at the TripComponent like it is implemented now.

Stops Component

The StopsComponent reads all stops from the GTFS stops.txt and adds a marker on the map for each of them, which when clicked shows the stop name, description and accessibility. This componenet is quite simple, but it shows how present a trasportation service is on a certain area. Removing the mocked GTFS and integrating the GtfsService with a rest API that could return stops from other agencies would make this component a lot more valuable.

Routes Component

The RoutesComponent takes all routes from the GTFS routes.txt and shows its detailed description and also offers a link to the source URL for the route.
